# Five o'clock — приглашение на чай

Одностраничный сайт-приглашение на день рождения с разделами: повод, программа, детали, настроение (фото), место и время, вишлист. Устроен как конструктор: **стили переключаются, тексты правятся прямо на странице, фото ставятся свои.**

## Файлы
| Файл | Что делает |
|---|---|
| `index.html` | Сам сайт |
| `styles.css` | Оформление и четыре стиля |
| `editor.js` | Конструктор (виден только вам) |
| `wishlist.js` | Логика вишлиста |
| `config.js` | Ключи Supabase — по желанию |
| `supabase.sql` | Одна таблица для общих отметок |

---

## Как редактировать

1. Откройте `index.html` (локально двойным кликом или уже опубликованный сайт).
2. Внизу справа — неприметная кнопка **«Редактировать»**. Нажмите. (Или добавьте `?edit=1` к адресу.)
3. Внизу появится панель:
   - **Стиль** — четыре точки: Селадон, Тушь, Бумага, Улун. Меняются мгновенно.
   - **Тексты** — кликайте прямо по любому заголовку или абзацу и печатайте.
   - **Фото** — наведите на любое изображение → «Сменить фото» → выберите файл. Три фото в «Настроении» и одно в «Месте». Картинки автоматически ужимаются.
   - **Ссылка карты** — вставьте ссылку на Яндекс.Карты или 2ГИС.
   - **Сбросить** — вернуть исходный вариант.
4. Когда всё нравится — **«Скачать сайт»**. Скачается готовый `index.html` со всеми вашими текстами, фото и выбранным стилем внутри.

> Правки живут в вашем браузере, пока вы не нажали «Скачать сайт». Гости увидят их только после того, как вы зальёте скачанный файл в репозиторий.

## Публикация на GitHub Pages

1. Создайте репозиторий (например, `five-oclock`).
2. **Add file → Upload files** → перетащите все файлы из этой папки → **Commit**.
3. **Settings → Pages** → Source: *Deploy from a branch* → ветка `main`, папка `/ (root)` → **Save**.
4. Через минуту сайт живёт по адресу `https://ВАШ-НИК.github.io/five-oclock/`.

Отредактировали → скачали новый `index.html` → загрузили его в репозиторий вместо старого. Сайт обновится сам.

## Вишлист: общие отметки (по желанию)

Без настройки вишлист работает, но отметка «Я дарю» видна только тому, кто её поставил. Чтобы гости видели чужие отметки:

1. Заведите бесплатный проект на **supabase.com**.
2. **SQL Editor → New query** → вставьте содержимое `supabase.sql` → **Run**.
3. **Project Settings → API** → скопируйте *Project URL* и ключ *anon public*.
4. Впишите их в `config.js` и залейте файл в репозиторий.

Готово: гости отмечают подарки, и занятое видно всем.

## Мелочи
- Кнопка «Редактировать» видна и гостям, но их правки останутся только в их браузере — на ваш сайт они не попадают. Если хочется убрать её совсем, удалите строку `<script src="editor.js" defer></script>` из `index.html` (тогда редактировать можно локально, а публиковать — версию без этой строки).
- Даты, время и адрес в шапке и в разделе «Место» — это просто текст, поправьте их в конструкторе.
- Свой домен вместо `github.io` подключается в **Settings → Pages → Custom domain**.
